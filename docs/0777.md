## 在 MySQL 中创建表

MySQL 表是结构化数据的集合，其中每一行代表一个事实，每一列代表该事实的属性。在创建表的过程中，每个列都被声明为保存特定的数据类型。

将一个表中的列数保持在 20 以下是一个很好的设计实践。因为列数的增加需要更多的 RAM 来在 select 查询执行期间创建临时表，而且这种需求随着列数的增加而呈几何级数增长。
